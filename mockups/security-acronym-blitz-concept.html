State 3 design notes — practice question shape:
  • Acronym in BIG monospace — instantly readable, the focal point of the screen. Reduces cognitive load.
  • SY0-701 domain pill below — gives implicit context (D1/D2/D3/D4/D5). Helps the user form mental categories.
  • Optional hint line in italic — for foundational difficulty only; suppressed on Hard difficulty (see Path B in dashboard tabs).
  • 4 answer cards — 1 correct + 3 plausible distractors. Distractor pattern: (a) close-but-wrong like "Security Operations, Analytics & Reporting" — sounds plausible but wrong order/word; (b) totally fabricated to be testably wrong; (c) acronym from another domain (e.g., for SOAR, "Standard Operating & Audit Routines" steals SOX-vibe). Standard CompTIA distractor style.

What does this acronym stand for?
SOAR
SY0-701 D4 · Security Operations
SOAR = Security Orchestration, Automation & Response. A SOAR platform ingests alerts from a SIEM (and other sources), applies analyst-authored playbooks, and automates routine actions (block IP, quarantine endpoint, open ticket, notify on-call). Common products: Palo Alto Cortex XSOAR, Splunk Phantom, IBM Resilient. Distractor (A) sounds plausible because it's also "S-O-A-R" but reverses orchestration and analytics — these are different capabilities; SOAR is about action, not reporting.
State 4 design notes — reveal & explainer:
  • Picked card turns red. Correct card turns green. Other distractors dimmed — keeps focus on the contrast.
  • Explainer is the killer feature: it gives the canonical definition + a one-line "what it does in real life" + why the distractor was wrong. The distractor explanation is the highest-value pedagogy. Reading it once is worth more than 3 MCQs.
  • "Add to study list" → enrolls into the existing v4.74.0 Spaced Repetition queue with 1-day interval. Acronyms graduate after 3 correct streak.
  • "Next →" auto-fires after 2.5s if the user does nothing (subtle — reduces friction in fast-paced drilling).

Distractor authoring discipline (the actual quality work):
  • Per acronym: 1 correct + 3 plausible distractors. Distractor patterns from CompTIA-style:
    • Close-but-wrong-word: SOAR = Security Orchestration, Automation & Response → distractor = Security Operations, Analytics & Reporting (same word count, sounds right).
    • Adjacent-domain: SOAR distractor = SOC Operating & Audit Routines (steals SOC vibe).
    • Pure-fabrication: Single Origin Authorization & Routing (sounds technical but isn't a thing).
  • Avoid distractors that are themselves valid acronym expansions for OTHER acronyms — too tricky, not testable on the real exam.
  • Foundational difficulty = include a hint line. Hard difficulty = no hint, distractors are ALL plausible adjacent terms.
Open decisions (push back if needed):
  • 7 categories at ~22 each vs 5 categories at ~30 each — denser categories = longer drill sessions but fewer mental jumps. 7-cat feels right for a beginner; 5-cat for cramming. Recommendation: 7-cat (more granular mastery tracking).
  • Reverse mode: in addition to "acronym → expansion", offer "expansion → acronym" mode. CompTIA tests both directions. Recommendation: ship reverse-mode in v4.91.1 once forward-mode validates the bank quality.
  • SR queue integration: missed acronyms auto-add to the v4.74.0 SR queue (per State 6). Already supported in scaffold; no new code.
  • Difficulty tiers: Foundational (hint visible) · Standard (no hint) · Hard (no hint + distractors more plausible). Existing scaffold already supports this. Default Standard.
