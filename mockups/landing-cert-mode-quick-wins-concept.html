4. Direct-to-diagnostic CTA. Primary purple button right under hero copy, replaces position currently held by the pass-pill (which moves slightly down or merges into the new CTA row). 6. Pass guarantee terms. Hand-wave for now — mockup shows the SHAPE ("don't pass first try → extra month free + extended access"). Exact terms (refund? extra Pro time? credit?) get firmed up when pricing is firmed up. Marketing-copy ship; no Stripe wiring needed yet.

7. Pass guarantee position. Standalone full-width section between cert grid + "Why CertAnvil". Sits in the funnel where visitors who scrolled past the cert grid are weighing commitment — the guarantee anchors the "what if I don't pass" anxiety just before they decide.

Network+ · N10-009 Domain 1.0 · Networking Concepts Exam Level

A network technician is troubleshooting slow file transfers between two servers. Which OSI-layer protocol is MOST likely to provide segment ordering and congestion control for the transfer?

Why TCP? TCP at Layer 4 provides reliable delivery, segment ordering (via sequence numbers), and congestion control (sliding window, slow start). IP (A) routes packets but is connectionless — no ordering, no congestion control. HTTP (C) is application-layer and uses TCP underneath, but isn't itself the protocol providing those properties. Ethernet (D) handles framing at Layer 2 — it doesn't see segments at all.
