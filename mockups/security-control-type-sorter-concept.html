Why "lighting" is Detective, not Deterrent

The trap: lighting seems to deter intruders. But CompTIA classifies the lighting itself as Detective — the camera/CCTV system uses the light to capture identifiable footage. Deterrents are explicit warnings (signs, dogs, security guards). The most reliable way to tell: ask "does this thing actively stop the attack? warn the attacker? or help us see?". Detective = help us see.

Implementation notes scope · schema · files · ship plan

Recommended ship plan

  • v1: MCQ mode only. Covers 80% of pedagogical value, ships in 1-2 days. The 6+4 button grid is mobile-friendly out of the box.
  • v2: Sort mode (drag-to-bucket). Adds depth for desktop users. Different ship — needs HTML5 drag-and-drop wiring + tablet/mobile fallback (long-press → tap-to-place). 2-3 days.
  • Mode toggle on dashboard makes both addressable from same drill entry. v1 ships with toggle UI but Sort tab disabled with "coming soon" pill.

Drill mechanics (MCQ mode)

  • Pool size: ~120 controls (20 per type × 6 types).
  • Question format: stem = control name + 1-line description + objective tag. Two answers required: 1 of 6 types + 1 of 4 categories. Submit only enabled when both axes locked in.
  • Scoring: per-control mastery on the COMBINED axis (24 possible cells × 5 controls ≈ 120 mastery slots). Mastered = 3 correct in a row. Wrong on either axis resets to 0.
  • Session length: 12 questions weighted toward weakest cells.
