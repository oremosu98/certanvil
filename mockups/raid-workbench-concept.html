Right. RAID 5 tolerates exactly one failed drive. The array keeps serving data, but its parity is spent, so a second failure would take it down.

Correct answer: Degraded. One drive down is within RAID 5's tolerance, so the array stays online, just with zero spare redundancy left.

Right. A degraded RAID 5 array rebuilds itself once the failed drive is replaced. No data was lost, so backups never enter the picture.

Correct answer: Hot-swap the failed drive and rebuild. The array is degraded, not failed. Its data is intact, and replacing the drive triggers an automatic rebuild.

Array math
RAID 5 usable capacity is (drives − 1) × size. Three 2TB drives give (3 − 1) × 2TB = 4TB. Fault tolerance for RAID 5 is a flat 1 drive, regardless of array size, so one failure degrades the array but never takes it offline.
