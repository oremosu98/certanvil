Design Decisions
WHY THIS SHAPE Mirrors what you saw on the real exam — broken topology with per-device MCQs to fix issues sequentially. Three items not five — keeps each PBQ ~3 min, scales to 30-50 total fixable issues across the catalog. Side panel (not modal) keeps spatial context with the topology.
VISUAL VOCABULARY Red ! badge with subtle pulse = unfixed issue. Green ✓ = correctly identified. Active device gets accent-purple ring. Broken cable for the disabled switch port renders as a red dashed line — visual confirmation when user looks at the topology after fixing.
CONFIG VIEWER Each broken device's config has the misconfigured field highlighted in red. Doesn't give the answer — the field is one of several possible problems — but it's a visible anchor for the user to reason about. Mirrors how Wireshark / Cisco show problem fields.
WHAT'S NOT IN THIS MOCKUP YET Production version would add: scoring readout (X/3 first-try correct, retries tracked), explanation breakdown after completion, "review your trace" mode showing all 3 issues + explanations side-by-side, integration with readiness analytics (PBQs as their own signal), and SR queue enrollment for missed concepts. All routine to add — the architecture's there.
SCOPE ESTIMATE (per the post-pass spec) Phase 0: this mockup. 1 day, done.
Phase 1: wire into TB infrastructure (schema + indicator overlay + MCQ panel + score readout). 3-5 days.
Phase 2: author the scenario catalog (~6 home + ~6 SOHO + 2-4 issues each = 30-50 fixable issues). Apply Jason Dion Method batching: ship as v4.86.x, v4.87.x. 2-3 weekends.
Phase 3: demo video for cert-mode launch (the "TikTok of TB in action" line item from cert_mode_launch_checklist.md). 1 day.
