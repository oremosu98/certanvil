Hop question — Step 2
What does SW-1 do with this frame?
L2 switching — MAC table lookup

A switch operates at Layer 2. It maintains a MAC address table mapping MACs → ports. When a frame arrives, it looks up the dst MAC and unicasts to that port. Flooding (option A) only happens when the MAC is NOT in the table (unknown unicast). Routing (option D) is what happens at the router on Step 3 — different layer entirely.

Hop question — Step 2
What does SW-1 do with this frame?
Layer confusion — Switch ≠ Router

You picked routing behaviour — but SW-1 is a Layer 2 switch. Switches forward frames based on destination MAC, full stop. They don't inspect IP headers, don't make routing decisions. The router (Step 3) is where IP-level routing happens — that's why the packet has TWO hops: switch first (L2), then router (L3). Internalise this and the rest of the trace clicks.

✗ Wrong · streak reset

Implementation notes scope · schema · files · biggest risk

Drill mechanics

  • Pool size: 10 curated scenarios at v1, ~4-6 hops each (≈ 47 total steps).
  • Question format: at each hop, MCQ with 4 options. Stem = "What does X do with this frame/packet?". Wrong = reveal explanation, trace continues. Right = reveal explanation, trace continues.
  • Mastery: per-scenario, only if ALL hops in a single run are correct. This is stricter than AB/CTS. Repeated wrong on the same hop counts toward weak-spot weighting.
  • Locking: scenarios unlock progressively. First 3 (ARP, cross-subnet routing, NAT) unlocked from start; rest unlock as earlier ones master. Forces a learning curve.
  • Resume: dashboard "Resume where you left off" picks up mid-scenario if user closed tab.

Biggest risk: SVG canvas authoring scale

  • 10 scenarios × ~5 hops × bespoke SVG layout per hop = potentially a lot of hand-authored network diagrams. Mitigation: build a shared SVG renderer that takes a JSON network description (devices, cables, subnet boxes, packet position) and generates the diagram. Each scenario is just data.
  • Renderer scope: ~6 device types (PC, laptop, switch, router, server, ?), 2 cable types (solid copper, dashed remote), subnet group boxes, animated packet position. Reuses TB device-glyph aesthetics for visual continuity.
  • Estimate: renderer = ~250 LOC of new code. Per-scenario data = ~50 LOC each × 10 = ~500 LOC. Total ~750 LOC for the canvas + scenarios.
