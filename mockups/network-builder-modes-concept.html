The firewall pair
In a screened subnet, two firewalls create an isolated DMZ segment. The outer firewall filters inbound traffic from the internet. The inner firewall protects the trusted LAN from the DMZ servers.

DMZ with screened subnet

Strengths
Subnet isolation is correct. The DMZ (172.16.0.0/24) is on a separate subnet from the trusted LAN (10.0.0.0/24), forcing all traffic through the firewall.
Gateway assignments are consistent. Every endpoint points to the correct router interface.
Concerns
FW-01 has no explicit deny rule. Without a trailing implicit deny, any traffic not matched by a permit ACE will pass through. Add deny ip any any as the last ACL entry.
No redundant path. A single link between Core-SW and FW-01 creates a single point of failure.
Upgrade suggestions
Add an IDS/IPS inline between the firewall and the DMZ segment for deep packet inspection of inbound HTTP traffic.
