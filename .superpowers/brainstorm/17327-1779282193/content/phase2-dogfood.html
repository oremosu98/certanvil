Phase 2 dogfood smoke · 8 steps

Run these on http://localhost:3131. If any step fails, ping back with which one + what you saw and I'll fix before we ship Phase 2.

01
Open TB v3 from sidebar
In the left sidebar, click Network Builder (the FIRST entry). Page opens full-viewport. Confirm the right-rail pill is now live — it should show a bronze Scenarios icon with no lock badge. (In Phase 1, the Scenarios icon was locked with a phase-hint tooltip. In Phase 2 it is active.)
02
Open the Scenarios picker
Click the Scenarios icon in the right-rail. A panel slides out from the right (320px wide). You should see 7 category headers and 8 scenario rows beneath them. The panel sits on top of the canvas — the right-rail pill (including any completion indicator) is hidden while the picker is open. Note: the picker does NOT auto-close when you pick a scenario — you close it manually with the × button.
03
Scan all 8 scenarios
Each row shows a title and N10-009 objective references. Verify the following categories and scenarios are present: Topology types (Star topology with central switch · Bus topology with hub) · Architectures · WAN · Cloud · Wireless · Security · VLAN / routing. If any row is missing or a category header is absent, flag it.
04
Pick “Star topology with central switch”
Click the Star topology with central switch row. The canvas populates: 1 switch + 1 server + 3 workstations + 4 cables. The intent chip in the status bar swaps to Lab · Star topology with central switch. Now close the picker using the × button — the right-rail reappears. The completion pill in the right-rail should read Goals met (green). If the pill says something else, check that all required devices are present on the canvas.
05
Delete a workstation — then restore it
With the picker closed, click any workstation on the canvas to select it, then press Delete. The completion pill in the right-rail should flip to 1 to go (dimmed). Now drag a workstation from the left palette back onto the canvas. The pill should flip back to Goals met (green). Critical: the picker must be closed for the completion pill to be visible.
06
Intent chip click → return to Free Build
Click the intent chip in the status bar (currently reads Lab · Star topology with central switch). The canvas clears. The intent chip changes to Free Build. The completion pill in the right-rail disappears (Free Build has no goals). You are now in a blank Free Build session.
07
Test backup-then-load
In Free Build, drag 3 random devices onto the canvas (any types you like). Open the Scenarios picker and pick any scenario other than the one you just left. A confirm dialog appears warning you that loading a scenario will replace your Free Build canvas. Click Accept (or equivalent). The new scenario loads. Click the intent chip to return to Free Build. Your original 3 Free Build devices should restore on the canvas. If they are missing, flag the backup-restore flow as broken.
08
Reload — state persists
While in Lab mode (any scenario loaded), reload the page. Navigate back to the Network Builder from the sidebar. The scenario should still be loaded — same devices, same intent chip, same completion pill state. Now click the intent chip to return to Free Build with your restored devices, then reload again. The restored Free Build canvas should persist. Critical: both the Lab state and the Free Build backup must survive a hard reload.
When all 8 pass: tell me “ship Phase 2” (or similar) and I’ll merge feat/tb-v3-phase2 into main, bump the version, expand the CLAUDE.md row, push to main, monitor CI, and prod-verify.

If any step is broken or feels off: tell me which step + what you saw + what you expected, and I’ll fix on the branch before we ship.

Known behaviour to be aware of: The Scenarios picker does not auto-close when you pick a row — you close it with the × button. The completion pill is only visible when the picker is closed (the picker panel covers the right-rail by CSS design). This is current shipped behaviour; auto-close on row-pick may be a follow-up if dogfood feedback asks for it.
